# 02 - 模型提供商配置

本文教你把 Claude Code 或 OpenCode 接入不同的「模型提供商（provider）」——也就是真正提供大模型服务的厂商或平台。读完后，你将能用官方 Anthropic、智谱 GLM、DeepSeek、MiniMax，甚至公司内网的私有化模型来驱动这两款工具。

> 前置阅读：如果还没装好工具，请先看 [01-安装与环境准备.md](01-安装与环境准备.md)；想深入理解配置文件结构，请看 [03-配置文件详解.md](03-配置文件详解.md)。

---

## 一、通用原理：接入任何模型只需「三要素」

无论哪一家提供商、用 Claude Code 还是 OpenCode，配置思路都一样，只需要三样东西（下文统称「三要素」）：

| 要素 | 含义 | 占位符 |
| --- | --- | --- |
| **base_url（端点地址）** | 模型 API 的服务入口地址，决定请求发往哪台服务器。 | `<BASE_URL>` |
| **token（密钥）** | 用来证明你身份的「钥匙」，也叫 API key。 | `<API_KEY>` |
| **model（模型 ID）** | 具体要用哪个模型，例如 `claude-opus-4-8`。 | `<MODEL_ID>` |

> 小贴士：Claude Code 原生说的是 Anthropic 协议（API 格式）。本文里的第三方提供商之所以能接入，是因为它们都提供了一个「**Anthropic 兼容端点**」——即一个长得和 Anthropic 官方接口一样、可以直接替换的地址。

### 全局配置 vs 项目级配置

「配置」放在哪，决定它的作用范围：

- **全局（用户级）配置**：对你电脑上所有项目生效。
  - Claude Code：`~/.claude/settings.json`（`~` 指你的用户主目录，Windows 下通常是 `C:\Users\你的用户名`）。
  - OpenCode：`~/.config/opencode/opencode.json`。
- **项目级配置**：只对当前项目生效，放在项目根目录下，会**覆盖/合并**全局配置中的同名项。
  - Claude Code：项目根目录的 `.claude/settings.json`。
  - OpenCode：项目根目录的 `opencode.json`。

> 配置层级的合并优先级、字段细节，见 [03-配置文件详解.md](03-配置文件详解.md)。

### 三种填法

本文每一家提供商都给出两套可复制示例：

1. **Claude Code —— settings.json 的 `env` 块**：写进配置文件，长期生效，推荐。
2. **Claude Code —— PowerShell 临时环境变量**：只在当前终端窗口生效，适合临时试用、不想改文件时。
3. **OpenCode —— `opencode.json`**：OpenCode 的统一配置方式。

## 二、Claude Code 的关键环境变量

接入第三方端点时，你会反复用到下面这些变量。先认识它们：

| 变量名 | 作用 |
| --- | --- |
| `ANTHROPIC_BASE_URL` | 端点地址（三要素里的 base_url）。 |
| `ANTHROPIC_AUTH_TOKEN` | 作为 **Bearer token** 发送的密钥。**第三方网关/提供商通常用这个。** |
| `ANTHROPIC_API_KEY` | Anthropic 官方风格的 API key 头。官方直连时用它；多数第三方更认 `ANTHROPIC_AUTH_TOKEN`。 |
| `ANTHROPIC_MODEL` | 主模型（干重活、需要高质量的模型）。 |
| `ANTHROPIC_SMALL_FAST_MODEL` | 小快模型（处理轻量、高频的后台任务，更便宜更快）。 |
| `CLAUDE_CODE_SUBAGENT_MODEL` | 子代理（subagent）所用的模型。 |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | 把 Claude Code 内部「Opus 档」映射到的实际模型。 |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | 把内部「Sonnet 档」映射到的实际模型。 |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | 把内部「Haiku 档（小快）」映射到的实际模型。 |

> **两种指定模型的方式**：一种是直接用 `ANTHROPIC_MODEL` / `ANTHROPIC_SMALL_FAST_MODEL` 指定主/小快模型；另一种是用 `ANTHROPIC_DEFAULT_OPUS_MODEL` / `..._SONNET_MODEL` / `..._HAIKU_MODEL` 把 Claude Code 内部的「Opus / Sonnet / Haiku 三档」分别映射到提供商的实际型号。**部分提供商（如智谱 GLM）官方推荐用后一种「按档映射」的写法**，下文各小节会按官方示例给出。

> **`ANTHROPIC_AUTH_TOKEN` 与 `ANTHROPIC_API_KEY` 的区别**：两者都是「填密钥」，但底层发送方式不同。`ANTHROPIC_AUTH_TOKEN` 以 HTTP `Authorization: Bearer <密钥>` 的形式发送；`ANTHROPIC_API_KEY` 走 Anthropic 原生的 `x-api-key` 头。不同提供商官方示例用的不一样（智谱用 `ANTHROPIC_AUTH_TOKEN`，DeepSeek/MiniMax 官方示例用 `ANTHROPIC_API_KEY`），下文按各家官方写法给出。**若用某一个报 401/鉴权失败，换另一个再试。**

> **自动填充模型列表**：当端点实现了 `/v1/models` 接口（私有化网关、部分代理常见），在 Claude Code 里输入 `/model` 选择器可以自动列出可用模型，省去手填。

## 三、如何在控制台找到三要素

不同平台「拿钥匙」的入口不一样。下表帮你定位（具体名称可能随官网改版变化，**字段以控制台为准**）：

| 提供商 | 在哪拿 API key | base_url（端点） | 模型 ID 示例 |
| --- | --- | --- | --- |
| Anthropic 官方 | console.anthropic.com → API Keys | 官方默认，无需手填 | `claude-opus-4-8` |
| 智谱 GLM | 智谱开放平台 → Coding Plan → API Keys | Claude Code 用 `https://open.bigmodel.cn/api/anthropic`；OpenCode 用 `https://open.bigmodel.cn/api/coding/paas/v4` | `glm-5.2[1m]`、`glm-5.1`、`glm-4.7`、`glm-4.5-air` |
| DeepSeek | DeepSeek 开放平台 → API keys | `https://api.deepseek.com/anthropic` | `deepseek-v4-pro`、`deepseek-v4-flash` |
| MiniMax | MiniMax 平台 → 账户/接口密钥 | `https://api.minimax.io/anthropic` | `MiniMax-M3`、`MiniMax-M2.7` |
| 内网私有化 | 由你公司平台分发 | 你的网关地址，如 `https://llm.内网域名/v1` | 由平台提供 |

## 四、Anthropic 官方（Opus 4.8）

官方有两条路径，二选一即可。

### 路径 A：订阅登录（最省心）

如果你有 Claude 的订阅（如 Pro/Max），无需任何配置，直接在终端运行：

```powershell
# PowerShell
claude
```

首次运行会弹出浏览器引导你登录授权。**预期结果**：登录成功后回到终端，即可直接对话，无需填 API key。

### 路径 B：使用官方 API key

适合用按量计费的 API key（在 console.anthropic.com 申请）。

**Claude Code —— settings.json 的 `env` 块**（写入 `~/.claude/settings.json`）：

```json
{
  "env": {
    "ANTHROPIC_API_KEY": "<API_KEY>",
    "ANTHROPIC_MODEL": "claude-opus-4-8",
    "ANTHROPIC_SMALL_FAST_MODEL": "claude-haiku-4-5-20251001"
  }
}
```

**Claude Code —— PowerShell 临时环境变量**（只在当前窗口生效）：

```powershell
# PowerShell
$env:ANTHROPIC_API_KEY = "<API_KEY>"
$env:ANTHROPIC_MODEL = "claude-opus-4-8"
$env:ANTHROPIC_SMALL_FAST_MODEL = "claude-haiku-4-5-20251001"
claude
```

**OpenCode —— `opencode.json`**（全局放 `~/.config/opencode/opencode.json`）：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-opus-4-8"
}
```

> OpenCode 读取标准的 `ANTHROPIC_API_KEY` 环境变量作为 Anthropic 的密钥；也可在配置里用 `provider` 块显式写 `apiKey`（见 [03-配置文件详解.md](03-配置文件详解.md)）。

- 模型 ID：主模型 `claude-opus-4-8`；小快模型可用 `claude-haiku-4-5-20251001`。
- 官方文档：https://docs.anthropic.com/en/docs/claude-code
- ⚠️ 模型 ID 与可用型号请以官方文档为准；**字段以控制台为准**。

## 五、智谱 GLM（Coding Plan）

> 本节按智谱官方文档刷新（2026-06）：
> [Claude Code 接入](https://docs.bigmodel.cn/cn/coding-plan/tool/claude) ｜ [OpenCode 接入](https://docs.bigmodel.cn/cn/coding-plan/tool/opencode)。
> **注意：Claude Code 与 OpenCode 用的是两个不同的端点**（见下），不要混用。

### 5.1 模型 ID 速览

| 型号 | 模型 ID | 说明 |
| --- | --- | --- |
| GLM-5.2（1M 上下文） | `glm-5.2[1m]` | 最新、最强，上下文窗口 100 万 token，适合做主模型。 |
| GLM-5.1 | `glm-5.1` | 上一代旗舰。 |
| GLM-4.7 | `glm-4.7` | Coding Plan 默认映射的 Opus/Sonnet 档型号。 |
| GLM-4.5-Air | `glm-4.5-air` | 轻量快模型，适合做 Haiku（小快）档。 |

> ⚠️ 型号会持续更新，请以[智谱控制台](https://bigmodel.cn/coding-plan)和官方文档显示的为准。`glm-5.2[1m]` 这种带 `[1m]` 后缀表示「百万上下文」变体。

### 5.2 Claude Code（端点：`/api/anthropic`，密钥用 `ANTHROPIC_AUTH_TOKEN`）

智谱官方推荐用「按档映射」的 `ANTHROPIC_DEFAULT_*_MODEL` 写法。

**settings.json 的 `env` 块**（官方推荐配置）：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "<API_KEY>",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5.2[1m]",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-5.2[1m]",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-4.5-air",
    "API_TIMEOUT_MS": "3000000",
    "CLAUDE_CODE_AUTO_COMPACT_WINDOW": "1000000"
  }
}
```

> 说明：`API_TIMEOUT_MS`（请求超时，毫秒）与 `CLAUDE_CODE_AUTO_COMPACT_WINDOW`（自动压缩上下文的窗口大小）是官方为「百万上下文 + 长任务」给出的推荐值，可按需调整或省略。想固定省钱跑 4.7，可把上面两个 `..._OPUS/SONNET_MODEL` 改成 `glm-4.7`。

**PowerShell 临时环境变量**：

```powershell
# PowerShell
$env:ANTHROPIC_BASE_URL = "https://open.bigmodel.cn/api/anthropic"
$env:ANTHROPIC_AUTH_TOKEN = "<API_KEY>"
$env:ANTHROPIC_DEFAULT_OPUS_MODEL = "glm-5.2[1m]"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL = "glm-5.2[1m]"
$env:ANTHROPIC_DEFAULT_HAIKU_MODEL = "glm-4.5-air"
claude
```

- **常见坑**：Claude Code 端点**必须**带 `/api/anthropic` 后缀，写成 `https://open.bigmodel.cn` 会返回 404。

### 5.3 OpenCode（端点：`/api/coding/paas/v4`，推荐用 `opencode auth login`）

OpenCode 走的是智谱 **Coding 专用端点**（OpenAI 风格，路径是 `/api/coding/paas/v4`），**不是** Claude 用的 `/api/anthropic`。官方推荐用交互式登录来填密钥：

```bash
# 终端
npm install -g opencode-ai
opencode auth login
# 在列表里选择 "Zhipu AI Coding Plan"，粘贴你的 API key
opencode
# 进入后用 /models 选择 GLM-4.7 / GLM-5.1 / GLM-5.2 等
```

对应的 `opencode.json`（项目根或 `~/.config/opencode/opencode.json`）只需声明端点，密钥由 `auth login` 管理：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "zhipuai": {
      "api": "https://open.bigmodel.cn/api/coding/paas/v4"
    }
  }
}
```

- 团队版（Team Plan）的 API key 与个人版不同，需用团队额度专用的 key。个人版入口：https://bigmodel.cn/coding-plan/personal/overview
- ⚠️ 端点、型号、字段以官方文档与控制台为准。

## 八、内网私有化部署（Anthropic 兼容网关）

公司内网常用 **LiteLLM**、自建代理、或 **vLLM** 等搭建一个「Anthropic 兼容网关」，把内部模型暴露成 Claude Code 能直接用的端点。

- **LiteLLM**：一个开源代理，能把各种模型统一成 OpenAI/Anthropic 兼容接口。
- **vLLM**：高性能的大模型推理服务，常用于自建部署。

**Claude Code —— settings.json 的 `env` 块**：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "<BASE_URL>",
    "ANTHROPIC_AUTH_TOKEN": "<API_KEY>",
    "ANTHROPIC_MODEL": "<MODEL_ID>",
    "NODE_EXTRA_CA_CERTS": "C:\\certs\\corp-root-ca.pem",
    "HTTPS_PROXY": "http://proxy.内网域名:8080",
    "HTTP_PROXY": "http://proxy.内网域名:8080"
  }
}
```

**Claude Code —— PowerShell 临时环境变量**：

```powershell
# PowerShell
$env:ANTHROPIC_BASE_URL = "<BASE_URL>"
$env:ANTHROPIC_AUTH_TOKEN = "<API_KEY>"
$env:ANTHROPIC_MODEL = "<MODEL_ID>"
# 自签证书：指向公司根证书（PEM 格式）
$env:NODE_EXTRA_CA_CERTS = "C:\certs\corp-root-ca.pem"
# 需要走代理时设置（不需要可省略）
$env:HTTPS_PROXY = "http://proxy.内网域名:8080"
$env:HTTP_PROXY = "http://proxy.内网域名:8080"
claude
```

**OpenCode —— `opencode.json`**：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "internal": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Internal Gateway",
      "options": {
        "baseURL": "<BASE_URL>",
        "apiKey": "<API_KEY>"
      },
      "models": {
        "<MODEL_ID>": {
          "name": "Internal Model"
        }
      }
    }
  },
  "model": "internal/<MODEL_ID>"
}
```

要点说明：

- **自签证书 `NODE_EXTRA_CA_CERTS`**：内网网关常用「自签 HTTPS 证书」。若不配置，连接会报证书校验失败（如 `unable to verify the first certificate`）。把变量指向公司根证书（`.pem` 文件）即可。
- **代理 `HTTPS_PROXY` / `HTTP_PROXY`**：如果访问网关需要经过公司代理，设置这两个变量；不需要代理则不要设置（否则反而连不通）。
- **`/v1/models` 自动填充**：若网关实现了 `/v1/models` 接口，在 Claude Code 中用 `/model` 选择器可自动列出网关上的可用模型，无需手填 `ANTHROPIC_MODEL`。
- ⚠️ 内网地址、模型 ID、证书路径均由你公司平台分发，**以控制台/平台说明为准**。

> 连接、证书、代理相关报错的排查，见 [09-最佳实践与排错.md](09-最佳实践与排错.md)。

## 九、模型选型建议（搭配省钱）

Claude Code 内部其实会用到三类「角色」的模型，合理搭配能在不牺牲体验的前提下省下不少费用：

| 角色 | 变量 | 选型思路 |
| --- | --- | --- |
| **主模型** | `ANTHROPIC_MODEL` | 干重活：写代码、复杂推理。选最强的（如 `claude-opus-4-8`、`glm-5.2[1m]`、`deepseek-v4-pro`、`MiniMax-M3`）。 |
| **小快模型** | `ANTHROPIC_SMALL_FAST_MODEL` | 处理高频小任务（如生成提交摘要、补全标题）。选便宜快的（如 `claude-haiku-4-5-20251001`、`glm-4.5-air`、`deepseek-v4-flash`）。 |
| **子代理模型** | `CLAUDE_CODE_SUBAGENT_MODEL` | 子代理批量、并行做搜索/分析。可用中档模型平衡成本与质量。 |

省钱搭配建议：

- 主模型用高质量大模型，小快模型务必换成便宜的快模型——后台调用很频繁，省下来的是大头。
- 子代理任务多为「检索 + 汇总」，对模型质量要求中等，可选比主模型便宜一档的型号。
- 试用阶段先用免费额度或便宜模型把流程跑通，再把主模型换成强模型。

> 更多省钱与排错技巧，见 [09-最佳实践与排错.md](09-最佳实践与排错.md)；想了解子代理、Skill 等扩展能力，见 [04-常用Skill与扩展.md](04-常用Skill与扩展.md)。
