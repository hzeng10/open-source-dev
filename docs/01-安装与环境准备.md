# 01 · 安装与环境准备

本篇带你从零搭好环境：安装 **Node.js**、**Git**，再安装 **Claude Code** 与（可选的）**OpenCode**，最后做一次最小验证，确认工具真的能用。

全程在 **Windows** 上演示。代码块开头会用注释标明运行环境（如 `# PowerShell` 或 `# Git Bash`），请在对应终端里执行。

---

## 1. 前置依赖

### 1.1 Node.js（LTS 版本）

> **Node.js** 是一个 JavaScript 运行环境。Claude Code 和 OpenCode 都通过 **npm**（Node 自带的包管理器）安装，因此必须先有 Node。建议安装 **Node 18 及以上**的 **LTS**（长期支持，更稳定）版本。

**安装方式一：用 winget（Windows 自带的包管理器，推荐）**

```powershell
# PowerShell
winget install OpenJS.NodeJS.LTS
```

**安装方式二：官网下载安装包**

打开 [https://nodejs.org/](https://nodejs.org/)，下载标有 **LTS** 的 Windows 安装包（`.msi`），双击按提示一路安装即可。

**安装完成后验证**（如果命令未识别，请关闭并重新打开终端再试）：

```powershell
# PowerShell
node -v
npm -v
```

预期输出示例（版本号可能不同，只要能正常显示即可）：

```text
v20.11.1
10.2.4
```

只要 `node -v` 显示 `v18.x.x` 或更高，就达标了。

---

### 1.2 Git for Windows

> **Git** 是版本控制工具，用来下载（克隆）开源项目、记录代码改动、提交 Pull Request。它几乎是开源开发的必备工具。

```powershell
# PowerShell
winget install Git.Git
```

安装完成后验证：

```powershell
# PowerShell
git --version
```

预期输出示例：

```text
git version 2.45.1.windows.1
```

---

### 1.3 终端选择

> **终端**是你输入命令的窗口。Windows 上有多种终端/外壳（shell）。

推荐组合：**Windows Terminal + PowerShell**。

- **Windows Terminal**：现代化的终端窗口，支持多标签、好看的字体，可从 Microsoft Store 安装。
- **PowerShell**：Windows 上功能强大的默认外壳，本指南大多数命令都用它。

此外，安装 Git for Windows 时会附带 **Git Bash**——一个类 Unix 的外壳。很多开源项目里的脚本是按 **POSIX**（类 Unix 系统标准）写的，遇到这类 `.sh` 脚本时，用 **Git Bash** 运行通常更顺畅。本指南会在代码块顶部用 `# Git Bash` 标明何时该用它。

## 2. 安装 Claude Code

### 2.1 安装

```powershell
# PowerShell
npm install -g @anthropic-ai/claude-code
```

> `-g` 表示「全局安装」，装完后可在任意目录直接使用 `claude` 命令。

### 2.2 验证安装

```powershell
# PowerShell
claude --version
```

预期输出示例（版本号可能不同）：

```text
1.0.0 (Claude Code)
```

### 2.3 首次运行与登录/认证

第一次启动 Claude Code：

```powershell
# PowerShell
claude
```

首次运行会引导你完成**认证（登录）**，主要有两种方式：

- **订阅登录**：用你的 Anthropic 账号（如 Claude 订阅）在浏览器中登录授权。
- **API key**：使用 Anthropic 的 API 密钥进行认证。

> **API key** 是一串身份凭证，相当于「访问模型服务的钥匙」，请妥善保管、不要泄露或提交到代码仓库。

具体如何获取 key、配置 `<API_KEY>` / `<BASE_URL>` / `<MODEL_ID>`，以及接入不同模型提供商的细节，见 [02-模型提供商配置.md](02-模型提供商配置.md)。

### 2.4 自检命令

Claude Code 提供了一个体检命令，能检查环境是否就绪、配置是否正常：

```powershell
# PowerShell
claude doctor
```

它会列出各项检查结果，若有问题会给出提示，按提示修复即可。

### 2.5 常见坑

**坑一：PowerShell 执行策略阻止脚本运行**

如果运行 `claude` 或 `npm` 相关脚本时报「无法加载……因为在此系统上禁止运行脚本」，是 PowerShell 的执行策略限制。执行下面这条命令放开当前用户的限制（安全且常用）：

```powershell
# PowerShell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

> `RemoteSigned` 表示本地脚本可运行、从网络下载的脚本需有签名，是兼顾安全与可用的常见设置。

**坑二：npm 全局 bin 目录不在 PATH 中**

如果 `npm install -g` 成功了，但输入 `claude` 提示「不是可识别的命令」，多半是 npm 的全局可执行文件目录没加入系统 **PATH**（系统查找命令的路径列表）。

先查看全局目录位置：

```powershell
# PowerShell
npm config get prefix
```

预期输出示例：

```text
C:\Users\<你的用户名>\AppData\Roaming\npm
```

把上面输出的路径添加到「环境变量 → Path」中：可在「开始菜单」搜索「编辑系统环境变量」，在用户变量的 `Path` 里新增该路径，保存后**重新打开终端**再试 `claude --version`。

## 3. 安装 OpenCode

> **OpenCode** 是开源的终端 AI 编码工具，可接入 75+ 模型提供商。它和 Claude Code 是两个独立工具，**可以共存**，互不影响。

### 3.1 安装

**通过 npm 安装**（包名为 `opencode-ai`，安装后命令为 `opencode`）：

```powershell
# PowerShell
npm install -g opencode-ai
```

也可以使用官方提供的**安装脚本**（在类 Unix 环境/Git Bash 中）：

```bash
# Git Bash
curl -fsSL https://opencode.ai/install | bash
```

> ⚠️ 版本/字段可能变化，请以官方文档为准：[https://opencode.ai/docs](https://opencode.ai/docs)（包名、命令名、安装方式以官网为准）。

### 3.2 验证与首次运行

```powershell
# PowerShell
opencode --version
```

预期输出示例（版本号可能不同）：

```text
0.1.0
```

进入任意一个项目目录后启动：

```powershell
# PowerShell
opencode
```

首次运行同样会引导你完成模型/认证配置，详见 [02-模型提供商配置.md](02-模型提供商配置.md)。

### 3.3 与 Claude Code 共存

Claude Code（命令 `claude`）和 OpenCode（命令 `opencode`）是不同的可执行程序，安装后各自独立，**可以同时存在于同一台机器**。你可以在不同项目里按需选择使用哪一个。

## 4. 验证安装（跑一个最小 prompt）

下面做一次端到端验证：建一个空目录，让 AI 工具帮你创建一个 `hello.txt` 文件，确认工具真的能干活。

**第一步：创建并进入一个空目录**

```powershell
# PowerShell
mkdir D:\dev\claude-test
cd D:\dev\claude-test
```

**第二步：启动工具并下达最小指令**

```powershell
# PowerShell
claude
```

启动后，在交互界面里输入这样一句 prompt：

```text
请在当前目录创建一个名为 hello.txt 的文件，内容写一行：Hello, Claude Code!
```

工具会请求你确认创建文件的操作（初学者直接允许即可）。完成后，验证文件是否生成：

```powershell
# PowerShell
type hello.txt
```

预期输出：

```text
Hello, Claude Code!
```

看到这行内容，说明 Claude Code 已经能正常读写文件、正常工作了。（若用 OpenCode，把上面的 `claude` 换成 `opencode`，流程一致。）
