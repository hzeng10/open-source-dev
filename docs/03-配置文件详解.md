# 03 - 配置文件详解

本文带你看懂 Claude Code 与 OpenCode 的核心配置文件：它们放在哪、各层级如何合并、关键字段是什么意思，并给出可直接复制的「最小可用」和「推荐」两套完整样例。

> 前置阅读：先装好工具见 [01-安装与环境准备.md](01-安装与环境准备.md)；想知道如何接入各家模型，见 [02-模型提供商配置.md](02-模型提供商配置.md)。

> 重要约定：本文所有 JSON 样例都是**合法 JSON**——JSON 不支持 `//` 注释，所以解释一律写在 JSON 代码块之外。复制时请勿在花括号里加注释。

# 第一部分：Claude Code 配置

## 一、配置文件层级与合并优先级

Claude Code 的设置分散在几个 `settings.json` 文件里。它们是**逐层合并**的：范围越小、越靠近项目本地的，优先级越高，会覆盖范围更大的同名字段。

| 文件 | 位置 | 作用范围 | 是否提交到 Git |
| --- | --- | --- | --- |
| **用户级** `settings.json` | `~/.claude/settings.json` | 你电脑上所有项目 | 不涉及（在主目录） |
| **项目级** `settings.json` | 项目根 `.claude/settings.json` | 仅当前项目，团队共享 | **应提交**（团队公用约定） |
| **本地级** `settings.local.json` | 项目根 `.claude/settings.local.json` | 仅当前项目，仅你本人 | **不应提交**（须 gitignore） |

> `~` 表示用户主目录，Windows 下通常是 `C:\Users\你的用户名`。

**合并优先级（从低到高）**：

```
用户级  <  项目级  <  本地级
```

也就是说：同一个字段，本地级会盖过项目级，项目级会盖过用户级。**预期效果**：你可以在用户级放通用偏好，在项目级放团队约定，在本地级放只属于你自己的密钥和临时开关。

## 二、主要字段讲解

下面逐个介绍最常用的字段。

### 2.1 `env`：环境变量

在 `env` 块里设置的变量，会在 Claude Code 运行时注入。常用于配置模型端点和密钥（详见 [02-模型提供商配置.md](02-模型提供商配置.md)）。

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "<BASE_URL>",
    "ANTHROPIC_AUTH_TOKEN": "<API_KEY>",
    "ANTHROPIC_MODEL": "<MODEL_ID>"
  }
}
```

### 2.2 `permissions`：权限控制

`permissions` 决定 Claude Code 可以自动做哪些操作，分三类列表：

- `allow`：命中规则的操作**自动放行**，不再询问。
- `deny`：命中规则的操作**直接禁止**。
- `ask`：命中规则的操作**每次询问**你确认。

规则写法形如 `工具名(参数模式)`，例如对 Bash 命令做精细控制：

```json
{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(npm run test:*)"
    ],
    "deny": [
      "Bash(rm -rf:*)"
    ],
    "ask": [
      "Bash(git push:*)"
    ]
  }
}
```

上例含义：自动允许查看 git 状态、diff 和跑测试；禁止 `rm -rf` 这类危险删除；推送代码前每次都问你。`:*` 表示「以此开头的任意后续参数」。

### 2.3 `model`：默认模型

直接指定默认主模型（也可用 `env` 里的 `ANTHROPIC_MODEL`，二选一即可）：

```json
{
  "model": "claude-opus-4-8"
}
```

### 2.4 `hooks`：钩子

`hooks` 让你在某些事件发生时**自动执行命令**（由 Claude Code 触发，而不是模型自己决定）。常见用途：每次编辑文件后自动格式化、提交前自动跑检查。

下面是一个最小例子：每次工具调用结束后，向终端打印一行字。

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo done"
          }
        ]
      }
    ]
  }
}
```

含义：当 `Edit`（编辑文件）工具用完后，运行 `echo done`。`matcher` 用来限定对哪个工具生效。

## 三、`CLAUDE.md`：项目记忆文件

`CLAUDE.md` 是放在项目里的纯文本/Markdown 说明文件，作用是把「这个项目的约定与上下文」长期告诉 Claude Code——比如技术栈、目录结构、代码风格、常用命令。每次在该项目里运行 Claude Code，它都会自动读取。

放置位置：

- **项目根目录** `CLAUDE.md`：最常见，描述整个项目的约定，建议提交到 Git 与团队共享。
- **子目录** `CLAUDE.md`：只对该子目录相关工作生效，用于局部约定。
- **用户级** `~/.claude/CLAUDE.md`：你的个人全局偏好，对所有项目生效。

> 你可以让 Claude Code 用 `/init` 命令自动为项目生成一份初始 `CLAUDE.md`。

---

## 四、MCP server 配置位置简介

**MCP（Model Context Protocol，模型上下文协议）** 让 Claude Code 连接外部数据源和工具（如数据库、浏览器、第三方 API）。MCP server 的配置通常放在项目根的 `.mcp.json`，或通过 `claude mcp add` 命令添加。

本文只点到为止，详细的 MCP、Skill、子代理用法见 [04-常用Skill与扩展.md](04-常用Skill与扩展.md)。

## 五、密钥安全

密钥（token / API key）一旦泄露，他人就能盗用你的额度。请务必：

- **不要把密钥提交进 Git 仓库**。尤其不要写进会被提交的项目级 `.claude/settings.json`。
- 把密钥放在以下任一安全位置：
  - **环境变量**（如临时设置 `$env:ANTHROPIC_AUTH_TOKEN`，见 [02-模型提供商配置.md](02-模型提供商配置.md)）。
  - **操作系统的凭据管理器**（Windows 凭据管理器等）。
  - **本地忽略文件** `.claude/settings.local.json`——它本就该被 gitignore。
- 在项目 `.gitignore` 中加入以下条目：

```gitignore
# Claude Code 本地配置（含密钥），不要提交
.claude/settings.local.json
```

> 如果不小心已经提交了密钥，应立即在对应平台「吊销/重置」该密钥，再重新生成。

## 六、完整 settings.json 样例

### 6.1 最小可用

只设定一个默认模型，其余走默认。适合刚上手。

```json
{
  "model": "claude-opus-4-8"
}
```

### 6.2 推荐

包含模型、端点密钥、常用权限和一个格式化钩子。把这份放在 `~/.claude/settings.json` 或项目 `.claude/settings.json`（密钥建议改放本地级文件）。

```json
{
  "env": {
    "ANTHROPIC_MODEL": "claude-opus-4-8",
    "ANTHROPIC_SMALL_FAST_MODEL": "claude-haiku-4-5-20251001"
  },
  "model": "claude-opus-4-8",
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(git log:*)",
      "Bash(npm run test:*)"
    ],
    "deny": [
      "Bash(rm -rf:*)"
    ],
    "ask": [
      "Bash(git push:*)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo formatted"
          }
        ]
      }
    ]
  }
}
```

> 把密钥（`ANTHROPIC_AUTH_TOKEN`/`ANTHROPIC_API_KEY`）单独放进 `.claude/settings.local.json` 的 `env` 块，避免随团队配置一起提交。

# 第二部分：OpenCode 配置

## 七、`opencode.json`：位置与合并语义

OpenCode 用单个 `opencode.json` 文件配置，分全局与项目两级：

| 文件 | 位置 | 作用范围 |
| --- | --- | --- |
| **全局** | `~/.config/opencode/opencode.json` | 所有项目 |
| **项目级** | 项目根 `opencode.json` | 仅当前项目 |

**合并语义**：OpenCode 的多层配置是**深度合并（merge）而非整体覆盖**。也就是说，项目级配置只会覆盖它显式写出的字段，其余字段仍沿用全局配置。**预期效果**：你可以在全局放好 provider 和密钥，在项目级只覆盖 `model` 一项即可。

---

## 八、主要字段讲解

| 字段 | 作用 |
| --- | --- |
| `$schema` | 配置文件的 JSON Schema 地址，填 `https://opencode.ai/config.json`，可让编辑器自动补全与校验。 |
| `provider` | 声明模型提供商。自定义 provider 用 `npm`（如 `@ai-sdk/anthropic` 或 `@ai-sdk/openai-compatible`）+ `options`（含 `baseURL`、`apiKey`、`timeout` 等）+ `models`。 |
| `model` | 默认模型，形如 `"提供商/模型"`，例如 `"anthropic/claude-sonnet-4-5"`。 |
| `agent` | 自定义代理（agent）的行为，如指定某个 agent 用哪个模型。 |
| `mcp` | 配置 MCP server，连接外部工具与数据源。 |

`provider.options` 里常用的 `timeout` 是请求超时（毫秒），网络慢或内网网关响应久时可调大。

## 九、完整 opencode.json 样例

### 9.1 最小可用

只指定一个默认模型（用 OpenCode 内置已知的 Anthropic 模型，密钥走 `ANTHROPIC_API_KEY` 环境变量）。

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5"
}
```

### 9.2 推荐

自定义一个 provider（指向任意 Anthropic 兼容端点）、设默认模型、配一个 agent 与一个 MCP server。

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "myprovider": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "My Provider",
      "options": {
        "baseURL": "<BASE_URL>",
        "apiKey": "<API_KEY>",
        "timeout": 60000
      },
      "models": {
        "<MODEL_ID>": {
          "name": "My Model"
        }
      }
    }
  },
  "model": "myprovider/<MODEL_ID>",
  "agent": {
    "build": {
      "model": "myprovider/<MODEL_ID>"
    }
  },
  "mcp": {
    "filesystem": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-filesystem", "."]
    }
  }
}
```

> 同样地，不要把含 `apiKey` 的项目级 `opencode.json` 提交进仓库；可改用全局 `~/.config/opencode/opencode.json` 或环境变量存放密钥。
