# 09 · 最佳实践与排错

本篇汇总用 **Claude Code** / **OpenCode** 时的高效使用习惯，以及遇到报错时的排查方法。读完前面几篇、能跑起来工具之后，本篇会帮你「用得更好、更省钱、更少踩坑」。

全程在 **Windows** 上演示。代码块开头会用注释标明运行环境（如 `# PowerShell` 或 `# Git Bash`），请在对应终端里执行。

---

## 1. 最佳实践清单

下面每一条都给出「**为什么**这样做」和「**怎么**做」，照着养成习惯即可。

### 1.1 先计划后执行：善用 plan mode

- **为什么**：让 AI 上来就动手改一大堆文件，方向错了很难收拾。先让它给方案，你确认后再执行，省时省心。
- **怎么做**：使用 **plan mode（计划模式）**——让 Claude Code 先输出「打算怎么做」的步骤清单，你看过、同意后它再真正动手。对于稍复杂的任务，先说「**先给我一个方案，不要直接改代码**」也能达到类似效果。

> **plan mode（计划模式）** 是 Claude Code 的一种工作模式：AI 只规划、只读不写，给出实施计划等你批准，避免它在你还没想清楚时就动手。

### 1.2 用 `CLAUDE.md` 固化项目约定

- **为什么**：每次都重复告诉 AI「本项目用 4 空格缩进、测试用 pytest、提交信息要英文」很累，也容易漏。
- **怎么做**：把这些**项目约定**写进项目根目录的 `CLAUDE.md` 文件，Claude Code 每次会自动读取并遵守。写法、放置位置见 [03-配置文件详解.md](03-配置文件详解.md)。

### 1.3 小步提交、勤跑测试

- **为什么**：一次改一大坨再提交，出了问题难定位；小步走则每步都可验证、可回退。
- **怎么做**：每完成一个小功能就 `git commit` 一次，并让 Claude Code 跑一遍测试确认没破坏现有功能。Git 提交流程见 [06-Git指南.md](06-Git指南.md)。

### 1.4 合理设置 permissions 的 allow 列表

- **为什么**：默认情况下，AI 每次要跑命令或改文件都会弹窗问你「是否允许」，频繁打断很烦。
- **怎么做**：把你信任的、反复出现的操作（如 `npm test`、`git status`）加入 **permissions（权限）的 allow（允许）列表**，AI 执行这些操作时就不再每次询问。配置方法见 [03-配置文件详解.md](03-配置文件详解.md)。注意别把危险操作（如删文件、推送代码）随意加入白名单。

> **permissions（权限）** 是 Claude Code 控制「哪些操作可以不经询问直接做」的设置；**allow 列表**就是其中的白名单。

### 1.5 密钥安全管理

- **为什么**：API key 相当于你账户的钥匙，一旦泄露（尤其是被提交进 Git 仓库并推到 GitHub），别人就能盗用你的额度，甚至产生费用。
- **怎么做**：
  - **绝不**把 key 写进代码或提交进仓库；把含 key 的文件（如 `.env`）写进 `.gitignore`（见 [06-Git指南.md](06-Git指南.md)）。
  - 用**环境变量**或系统**凭据管理器**保存 key。设置环境变量示例：

```powershell
# PowerShell（仅当前窗口有效）
$env:ANTHROPIC_AUTH_TOKEN = "<你的_API_KEY>"
```

  - 详细的密钥与 `<BASE_URL>` / `<MODEL_ID>` 配置见 [02-模型提供商配置.md](02-模型提供商配置.md)。

### 1.6 模型选型省钱

- **为什么**：强模型聪明但贵，弱模型便宜但能力有限。所有任务都用最强模型很烧钱。
- **怎么做**：**主模型用强模型**（处理需要推理、把控全局的主线任务）；**子代理（subagent）和小而快的任务用便宜的模型**（如批量改名、简单格式整理）。如何配置不同模型见 [02-模型提供商配置.md](02-模型提供商配置.md)。

### 1.7 让 AI 解释，而不是盲目接受

- **为什么**：盲目接受 AI 的改动，可能引入你看不懂、也维护不了的代码或 bug。
- **怎么做**：对关键改动，让它先**解释思路**（「为什么这么改？」）；接受前**逐行审阅 diff**（差异对比，见 [08-IDE集成指南.md](08-IDE集成指南.md) 的内联 diff）。你是最终负责人，AI 是助手。

### 1.8 上下文管理

- **为什么**：对话越长，AI 要记住的「上下文」越多，不仅变慢、变贵，还可能被无关的旧内容干扰、答偏。
- **怎么做**：
  - 一个任务做完、要开始**无关的新任务**时，用斜杠命令 `/clear` 清空当前会话上下文，轻装上阵。
  - 把需要**长期记住**的背景（项目约定、目录结构、常用命令）写进 `CLAUDE.md`，而不是每次靠对话临时交代（见 [03-配置文件详解.md](03-配置文件详解.md)）。

> **上下文（context）** 指 AI 当前对话中「记着」的所有信息；`/clear` 是清空它的斜杠命令。

## 2. 排错表

遇到报错先别慌，对照下表「现象 → 可能原因 → 解决办法」逐项排查。下文涉及的 `<BASE_URL>` / `<MODEL_ID>` / 各环境变量配置细节，统一见 [02-模型提供商配置.md](02-模型提供商配置.md)。

| 现象 | 可能原因 | 解决办法 |
| --- | --- | --- |
| **401 Unauthorized** / **403 Forbidden**（未授权 / 被拒绝） | token 填错；`ANTHROPIC_AUTH_TOKEN` 没设置；`<BASE_URL>` 与 key 不匹配（用了 A 家的 key 却连到 B 家的地址） | 检查 key 是否正确、有没有多余空格；确认 `ANTHROPIC_AUTH_TOKEN` 已设置；核对 base_url 和 key 来自**同一个提供商** |
| **404 Not Found**（找不到端点） | API 端点路径写错，少写了路径后缀 | 核对完整 base_url。**例**：智谱（GLM）必须用 `https://open.bigmodel.cn/api/anthropic`，若漏掉结尾的 `/api/anthropic` 就会 404 |
| **模型名不被识别** / `model not found` | `ANTHROPIC_MODEL` 或 `<MODEL_ID>` 写错，模型名拼错或该提供商没有这个模型 | 登录提供商的**控制台**，按其「模型列表」里的**准确名称**填写 `<MODEL_ID>` |
| **TLS / 证书错误**（`self-signed certificate`、`unable to verify` 等） | 私有化部署用了**自签证书**（自己签发、未被系统默认信任的证书），Node 不认 | 设置环境变量 `NODE_EXTRA_CA_CERTS` 指向证书文件（见 [02-模型提供商配置.md](02-模型提供商配置.md)） |
| **请求超时 / 连不上**（`timeout`、`ECONNRESET`、`ETIMEDOUT`） | 需要走代理才能访问；或网络慢、默认超时太短 | 设置 `HTTPS_PROXY` / `HTTP_PROXY` 指向你的代理；或调大超时时间 |
| **`claude` / `opencode`：不是可识别的命令** | npm 全局 bin（可执行文件）目录不在系统 **PATH** 中 | 用 `npm config get prefix` 查路径，把它加入环境变量 Path，重开终端（详见 [01-安装与环境准备.md](01-安装与环境准备.md)） |
| **PowerShell 报「禁止运行脚本」** | PowerShell 执行策略限制了脚本运行 | 运行 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`（见下方代码） |
| **中文乱码 / 换行符（CRLF）问题** | Windows 与类 Unix 换行符不一致；`core.autocrlf` 设置不当 | 按团队约定设置 Git 的 `core.autocrlf`（见 [06-Git指南.md](06-Git指南.md)） |

**TLS 证书：设置 `NODE_EXTRA_CA_CERTS`**

```powershell
# PowerShell（仅当前窗口有效）
$env:NODE_EXTRA_CA_CERTS = "C:\path\to\your-ca.crt"
```

**代理：设置 `HTTPS_PROXY` / `HTTP_PROXY`**

```powershell
# PowerShell（仅当前窗口有效，端口换成你代理软件的实际端口）
$env:HTTPS_PROXY = "http://127.0.0.1:7890"
$env:HTTP_PROXY  = "http://127.0.0.1:7890"
```

**PowerShell 执行策略：放开当前用户限制**

```powershell
# PowerShell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

> `RemoteSigned` 表示本地脚本可运行、从网络下载的脚本需有签名，是兼顾安全与可用的常见设置。

> 上面用 `$env:` 设的环境变量**只对当前终端窗口有效**，关掉就没了。若想长期生效，请在「编辑系统环境变量」里加入用户变量（见 [01-安装与环境准备.md](01-安装与环境准备.md) 的 PATH 设置方法）。

### 2.1 用 `claude doctor` 自检

排查问题时，第一件事可以先跑 Claude Code 自带的「体检」命令：

```powershell
# PowerShell
claude doctor
```

它会逐项检查环境、配置、网络等是否就绪，并对有问题的项给出提示，按提示修复即可。

> ⚠️ 各提供商的端点地址、模型名、认证方式可能随时调整，遇到与本文不符的情况，请以对应官方文档为准。

## 3. 获取帮助

当本指南没覆盖到你的问题时，按下面的顺序求助：

- **`/help` 斜杠命令**：在 Claude Code 交互界面里输入 `/help`，查看可用命令与内置帮助。
- **官方文档**：
  - Claude Code：[https://docs.anthropic.com/en/docs/claude-code](https://docs.anthropic.com/en/docs/claude-code)
  - OpenCode：[https://opencode.ai/docs](https://opencode.ai/docs)
- **社区**：官方文档通常会链接到 GitHub Issues、Discord 等社区渠道。搜索你的报错关键词，往往能找到别人遇到过的同样问题与解法。

> ⚠️ 工具更新较快，部分命令、字段、报错文案可能与本文略有出入，请以官方文档为准：[Claude Code 官方文档](https://docs.anthropic.com/en/docs/claude-code) ｜ [OpenCode 官方文档](https://opencode.ai/docs)
